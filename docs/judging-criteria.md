# Judging Criteria

## Overview

Games are judged on creativity, execution, and how well contestants leveraged AI-assisted development.

## Categories (Equal Weight)

### 1. Creativity & Originality (25%)
- How unique is the gameplay concept?
- Did the contestant go beyond the template?
- Are there surprising or innovative mechanics?
- Does the game have a distinct identity?

### 2. Gameplay & Fun Factor (25%)
- Is the game enjoyable to play?
- Are the mechanics well-designed?
- Is there appropriate challenge/progression?
- Does the game feel complete?

### 3. Technical Execution (25%)
- Does the game run without bugs?
- Is the code well-structured?
- Are systems properly implemented?
- Is the architecture clean and maintainable?

### 4. AI-Assisted Innovation (25%)
- How effectively was AI used?
- Did the contestant maximize their limited credits?
- Is there evidence of smart prompt engineering?
- How much was accomplished with the credit budget?

## Scoring Scale

| Score | Meaning |
|-------|---------|
| 5 | Exceptional — goes far beyond expectations |
| 4 | Strong — polished and well-executed |
| 3 | Good — solid implementation with some gaps |
| 2 | Fair — basic but functional |
| 1 | Needs work — incomplete or broken |

## Tips for High Scores

### Creativity
- Don't just tweak numbers — redesign systems
- Combine mechanics in unexpected ways
- Give your game a unique theme and identity

### Gameplay
- Playtest your game multiple times
- Ensure there's a clear goal and feedback
- Balance difficulty — not too easy, not impossible

### Technical
- Keep code modular and clean
- Handle edge cases (empty inventory, zero HP, etc.)
- Test the full game loop (start → play → end)

### AI Innovation
- Document your AI workflow in your README
- Show how you used prompts efficiently
- Demonstrate creative use of limited credits

## Bonus Points

- Procedural content generation
- Multiple endings or paths
- Accessibility features
- Original art or sound (even simple)
- Innovative use of WinForms capabilities
