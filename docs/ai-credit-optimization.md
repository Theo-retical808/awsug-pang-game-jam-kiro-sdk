# AI Credit Optimization

## You Have ~50 Credits. Make Them Count.

Each interaction with AI costs credits. This guide helps you maximize what you build.

## Credit Budget Strategy

| Phase | Credits | Goal |
|-------|---------|------|
| Core mechanics | 15-20 | Build your unique gameplay |
| Content | 10-15 | Generate levels, dialogue, data |
| Polish | 5-10 | UI, balance, bug fixes |
| Reserve | 5-10 | Unexpected issues |

## High-Value Prompts (Best ROI)

These prompts generate the most code per credit:

1. **Data generation** — "Generate 20 enemy types as JSON" (lots of content, 1 credit)
2. **New system** — "Create an inventory system in Items/Inventory.cs" (whole feature, 1 credit)
3. **Batch changes** — "Add Speed, Luck, and Crit stats to Unit.cs and update InfoPanel" (multiple changes, 1 credit)

## Low-Value Prompts (Avoid)

These waste credits:

1. "Explain how the grid system works" — Read the code instead
2. "What should I add to my game?" — Decide yourself
3. "Fix the formatting" — Do it manually
4. "Rename variables" — Use IDE refactoring

## Architecture Advantages

This SDK is designed to save you credits:

- **Small files** — AI can edit one file without reading 10 others
- **Clear naming** — You know where things are without asking
- **Modular systems** — Add features without touching existing code
- **JSON data** — Generate content without code changes
- **Predictable patterns** — AI understands the structure quickly

## Workflow: Maximum Efficiency

1. **Plan first** (0 credits) — Decide what to build before asking AI
2. **Read the code** (0 credits) — Understand what exists
3. **Write specific prompts** (1 credit each) — Get results in one shot
4. **Test immediately** — Catch issues before spending more credits
5. **Iterate only if needed** — Don't polish prematurely

## Emergency Credit Conservation

Running low? These techniques help:

- Write simple code yourself (variable changes, number tweaks)
- Copy patterns from existing files manually
- Use JSON files for content instead of code changes
- Focus remaining credits on the hardest problems
- Skip nice-to-have features, focus on core gameplay
